Longform template

A boilerplate for longform journalism articles. Made for designers learning responsive webdesign and developers wanting a streamlined workflow.


By James Benedict
Last Updated July 23, 2016

This template is meant for people of all skill levels and aims to allow designers to spend more time designing and less time coding. Tweaks and changes are encouraged as it is only the boilerplate for you to build on. You can view some examples of stories developed with earlier versions of the template here.

Alignment

link to stand alone example

There are nine layout classes to get started with and the css is found in alignment.css.

Article

The article tag creates the main column on the screen and is where this text is now. It takes up 50% of the screen, with a 25% margin on either side. To go wider use the wide or full classes. You can also divide the article column up internally with aside.

Asides

There are three type of asides, two that go inside the article and one that does not. For each type, there are two sepearte classes for right and left. The two article are aside class="left" and aside class="left-wide". The aside class="left" placed outside of the article column will float to the edge of the screen.

Wide

The wide class is usually used for media or graphics. It has a width of 75% and a margin-left of 15%. You can either place a figure inside a wide div or apply the class to the figure itself.

full

The full class is a fullscreen alignment best used for media.

Development

I wanted to write a little about my development process to show people getting started some tools to speed up their coding. Don't get caught up trying to match this exactly, and if something gets confusing come back to it later. These just improve your efficiency they don't add anything new.

Tools

Sublime Text is a text editor with too many useful features to explain here. This walkthrough will get you up and going. I recommend installing the emmet plugin as well.

Sass describes itself as css with superpowers and will make your style sheets easier to work with. It has a ton a features like variables and nesting, but if you want to use them you need to send your sass files through a preprocessor to convert them back to css for your webpage. There are a few options to do this but my favorite is gulp.

gulp is a javascript library that automatically does tasks you don't want to do yourself. Want your browser to reload when you change a style? Want minifyed images? Want to convert your sass files to css? Gulp does all these and more with a single command line. This tutorial should get you up and running.
